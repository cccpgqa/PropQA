# Dataset schema

Each propagation record contains `Source` and `Infestor` objects. `Infestor`
is a legacy field name retained for compatibility and denotes the downstream
target artifact.

Common artifact fields include:

- `project`: normalized repository identifier.
- `type`: `pr`, `commit`, or `issue`.
- `number`, `pr_number`, or `commit_sha`: artifact identifier.
- `title`, `body`, or `message`: textual change description.
- `created_at`, `merged_at`, and `updated_at`: available timestamps.
- `html_url`: canonical GitHub URL.
- `base_commit_sha` and `merged_sha`: revisions used to reconstruct source or
  target repository states when available.
- `file_names_all`: all changed files retained from the GitHub artifact.
- `primary_file_names`: files used as primary localization ground truth.
- `auxiliary_file_names`: tests, fixtures, generated artifacts, or supporting
  files retained separately.

The detection dataset additionally records curation and ground-truth metadata.
Statement ground truth is reconstructed from target patch hunks and includes
only statements present in the target pre-fix revision that are subsequently
edited or deleted.

The practical validation dataset contains stable `case_id` values from
`CASE-001` through `CASE-014`. They join pair records with
`results/practical_localization_results_14.json`. These cases are independent
of the detection benchmark and represent retrospectively confirmed downstream
changes.

`results/practical_history_candidates.json` is generated by searching from
upstream changes into later target-repository history. Each source entry
records the target snapshot, PropQA file and statement predictions, QA traces,
and ranked later commits with file, patch, semantic, and statement evidence.
These candidates are not ground truth until manually validated.
